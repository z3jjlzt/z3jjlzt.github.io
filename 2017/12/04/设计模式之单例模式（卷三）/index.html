设计模式之单例模式。

确保系统中只有唯一一个实例，当这个唯一实例创建成功之后，我们无法再创建一个同类型的其他对象，所有的操作都只能基于这个唯一实例。

主要步骤

  1. 构造器私有化。
  2. 定义一个private私有的变量。
  3. 提供一个public类方法让外界访问唯一的实例。

0x00 饿汉式

饿汉式在类加载的时候就已经创建了实例，线程安全，增加了类加载时间。

0x01 懒汉式（延迟加载）

在第一次使用的时候调用方法的时候创建实例，在多线程情况下可能产生多个实例，需要使用双重锁保证线程安全。

需要注意的是，即使使用了双重检查，也不能保证一定正确处理，根本原因是因为初始化一个对象并使一个引用指向他 这个过程不是原子的，这其中一旦发生指令重排序，就有可能抛出NPE，引用了一个未初始化的实例对象，如 A a = new A();并不是原子操作。解决的方法是在字段中加volatile，且必须在jdk1.5之后，因为之前的版本volatile还不完善。详情参考jsr-133。

A a = new A()这个操作可以拆成：

  1. 栈内存开辟空间给a引用
  2. 堆内存开辟空间准备初始化对象
  3. 初始化对象
  4. 栈中引用指向这个堆内存空间地址
    当发生重排序时，如果为1243，在多线程环境下则4指向了一个未初始化的实例，这块堆内存还没初始化就直接返回，导致了引用null。

顺便讲讲Java创建实例对象的过程。在A a = new A();中。

  • A a为变量的声明，表示将变量名称与对象类型关联的变量声明；
  • 初始化，三种执行对象初始化的结构，分别是 实例变量初始化、实例代码块初始化 以及 构造函数初始化。

0x10 静态内部类

原理是使用JVM虚拟机本身的特性保证；既虚拟机会保证一个类的clinit方法会在多线程环境下被正确的加锁、同步，可以参考之前的虚拟机类加载机制篇章。

0x11 枚举

枚举还可以设计成多实例方式。
